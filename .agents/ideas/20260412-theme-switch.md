# テーマ切り替え機能 実装アイデア

## ゴール
ユーザーがアプリ紹介LPのライトモード/ダークモードを手動で切り替えられるようにする。システム設定への追従も維持する。

## 課題
- 既存のCSSはメディアクエリ (`@media (prefers-color-scheme: dark)`) を使ったシステム依存のスタイル切り替えになっている。
- 手動切り替えを実現するには、HTMLタグ等に属性（例：`data-theme="dark"`）を付与し、それに追従するようにCSSを書き換える必要がある。

## 解決策・構成
1. **CSSの変数化**
   - 既存の直接記述されているダークモード固有の設定を、カスタムプロパティ（CSS変数）として抽出する。
   - 例: `--header-bg`, `--mockup-shadow`, `--icon-wrap-bg` など
   - CSS内で `@media (prefers-color-scheme: dark)` を使用していた箇所を削除し、`:root[data-theme="dark"]` による変数上書きのみで管理するようにリファクタリング。

2. **ちらつき防止 (FOUC 対策)**
   - 各HTMLファイルの `<head>` 末尾に、初期描画前（パース中）に `localStorage` およびシステム設定からテーマを割り出し、`<html>` に `data-theme` 属性をすぐ反映させるインラインスクリプトを追加する。

3. **トグルボタンの追加**
   - ヘッダー (`header__nav`) 内に、テーマ切り替えボタン（太陽と月のSVGアイコン）を配置する。
   - `data-theme` 属性の状態によって、太陽・月のどちらかを表示する。

4. **JavaScript による切り替えロジック**
   - `script.js` にて、テーマ切り替えボタンのクリックイベントを取得し、`data-theme` のトグルと `localStorage` への保存を行う関数を実装。
   - システム側のカラースキーム変更時のイベントリスナー (`matchMedia`) も追加し、`localStorage` のユーザー設定が存在しない場合は自動追随させる。
