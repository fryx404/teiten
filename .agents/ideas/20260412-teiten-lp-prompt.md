# Teiten - LP自動生成用プロンプト仕様書

このドキュメントは、別セッションの生成AI（ChatGPTやClaudeなど）に「定点観測アプリ Teiten」のランディングページ（LP）をコーディングさせるための指示書です。以下の【AIへの指示文】をコピー＆ペーストするだけで、高品質なLPのベースコード（HTML/CSS/JS）が出力されるように設計されています。

---

## 🤖 【AIへの指示文】（ここからコピー）

あなたは優れたWebフロントエンドエンジニアであり、卓越したUI/UXデザイナーです。
1日1枚の写真を記録するミニマルでポップな定点観測アプリ「Teiten（定点）」のランディングページ（LP）を、GitHub Pages用の静的ファイル（HTML/CSS/JS）として作成してください。
以下の仕様と要件に厳密に従って、コード全体を生成してください。

### 1. プロジェクト概要
- **アプリ名**: Teiten（定点）
- **コンセプト**: 1日1枚の写真を記録し、ミニマルかつポップな世界観で振り返る「定点観測アプリ」。
- **キャッチコピー**: 「何気ない今日を、一枚の定点に。」
- **ターゲットと価値**: 毎日変化する日常を簡単に記録したい人。クラウド同期やアカウント登録の煩わしさがない、自分だけのオフライン写真日記。

### 2. デザインシステム (fryx404 Pop)
個人サイト（fryx404）の世界観を踏襲した、クリーンでポジティブかつリッチなデザインにしてください。「単なるMVP」に見えない、ワクワクするようなプレミアム感を求めます。

- **テーマ**: OS設定に連動するライト / ダークモードの両対応（CSSの `@media (prefers-color-scheme: dark)` を使用）。
- **カラーパレット**:
  - **Light Mode**: 
    - Background: `#FFFFFF`
    - Surface: `#FFFFFF`
    - Primary: `#00D1FF` (Electric Cyan - アクセントやボタンに。グラデーション活用も可)
    - Accent: `#FFD600` (Sunshine Yellow - ハイライトや強調テキストに)
    - Text: `#2D3436`
  - **Dark Mode**:
    - Background: `#000000`
    - Surface: `#1A1A1A`
    - Primary: `#00D1FF`
    - Accent: `#FFD600`
    - Text: `#F5F5F5`
- **デザイン原則**: 
  - **角丸**: `border-radius: 16px;` またはそれ以上を多用し、フレンドリーでポップな印象に。
  - **シャドウ**: ライトモードでは水色系の薄いドロップシャドウで浮遊感を演出、ダークモードでは不透明度を抑えた微弱なシャドウを使用すること。
  - **タイポグラフィ**: Google Fontsから `Inter` や `Outfit`、日本語フォントとして `Noto Sans JP` などを読み込み、太めで視認性の高いフォントウェイトを使用すること。
  - **マイクロアニメーション**: ボタンのホバー時にフワッと浮き上がるような滑らかなトランジション（`transition: all 0.3s ease;`等）を全インタラクティブ要素に追加すること。

### 3. LPの構成（セクション順）
シングルページの構成とし、上から順に以下の要素を実装してください。

1. **Header (ナビゲーション)**
   - 左端: アプリアイコン（または太字の「Teiten」テキストロゴ）。
   - 右端: 「Privacy Policy」「Terms」へのリンク（小文字またはさりげないデザインで）。
2. **Hero セクション**
   - 大きく太いキャッチコピー「何気ない今日を、一枚の定点に。」とサブテキスト。
   - ダウンロードを促すCTAボタン（「Google Playで手に入れる」等。バッジ画像プレースホルダーで可）。
   - アプリ画面のスマホモックアップが浮遊しているようなレイアウト。PCでは右側、スマホでは下部に配置。アニメーション（上下にゆっくりフワフワ動くなど）を追加すること。
3. **Features (機能紹介)**
   - カード型のレイアウトで3つの特徴を表現。ホバーエフェクトをリッチにすること。
   - 特徴1: **「1日1枚、迷わない」** (カレンダービューで日々の変化が一目でわかる)
   - 特徴2: **「一言を添える」** (写真にテキストカードをつけて、感情を記録)
   - 特徴3: **「安全・プライベート」** (完全ローカルファースト。アカウント登録なし、完全無料、データはデバイス内のみに保存)
4. **Bottom CTA (Call to Action)**
   - ページ下部を飾るもう一つのハイライトエリア。再度アプリのダウンロードを促すブロック。背景色にPrimaryカラー（#00D1FF）を大胆に使うなどの工夫をすること。
5. **Footer**
   - コピーライト `© 2026 Takumi Furuya (fryx404)`
   - 既存の「利用規約」「プライバシーポリシー」へのリンク（現状のGitHub Pagesにあるため、相対パスで `./terms` と `./privacy` など該当ファイルへリンクするように仮置きしておく）。

### 4. 実装要件・出力形式
- **利用技術**: HTML5, Vanilla CSS, Vanilla JavaScript (複雑なビルドツール、React等のフレームワーク、TailwindCSSなどのユーティリティCSSは原則使用せず、プレーンなコードを使用すること)。
- **レスポンシブ**: モバイルファーストで実装し、メディアクエリを使用してタブレット・PC（`max-width: 1200px`程度）でも中央揃えで美しく表示されるようにすること（Flexbox/CSS Gridを駆使すること）。
- **SEO・メタタグ**: `<head>`内に、適切な `<title>`, `<meta name="description">`, `viewport` などを設定すること。
- **納品物のお願い**: コピペで1発動作する `index.html` （CSS/JSは別ファイル `style.css`, `script.js` に分けるか、すべてインラインで1ファイルにするか明記し）のコードを出力してください。画像パスは `./assets/` などのプレースホルダーを使用し、必要な箇所のコメントを残してください。

## 💡 この指示書の使い方

1. 上記の【AIへの指示文】を丸ごとコピーします。
2. ChatGPT (GPT-4o) や Claude 3.5 Sonnet など、コード生成に優れたAIにペーストして実行します。
3. AIが生成したコード（HTML/CSS等）を、Teitenリポジトリの `docs/` ないしは GitHub Pages の公開対象ディレクトリに配置します。
4. プレースホルダーとなっている画像部分（スマホモックアップやGoogle Playバッジ）を、実際のアセットに差し替えます。
5. 既存のMarkdown規約ファイル（Terms, Privacy）へ適切にリンクされるよう微調整します。
